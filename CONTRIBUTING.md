# Contributing

Issue reports welcome. Drop a real-world test case or a loophole you found — those are the most useful contributions.

For skill edits: fork, propose a change, explain the failure mode it fixes. Skills follow a strict TDD discipline (RED baseline → GREEN implementation → REFACTOR via subagent pressure-test). See the individual skill directories for audit trails.

Not accepting new skills from contributors this year — the series is opinionated by design. That may change later.
